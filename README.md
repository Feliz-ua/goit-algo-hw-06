### Мета роботи 
- Створити та візуалізовувати граф — модель певної мережі.
- Провести аналіз основних характеристик.
- Проаналізувати дію алгоритмів DFS і BFS для знаходження шляхів.
- Реалізувати алгоритм Дейкстри для знаходження найкоротшого шляху в розробленому графі.

### Виконання

За допомогою бібліотеки NetworkX створено неорієнтований граф міських доріг з кількістю вершин (об'єктів у місті) — 8, з кількістю ребер — 12 (дороги між об'єктами з довжиною у кілометрах):
- "Центр міста"
- "Залізничний Вокзал"
- "Мій дім"
- "Місце роботи"
- "Ринок"
- "Торгівельний центр"
- "Парк"
- "Річковий вокзал"

Програма моделює транспортні маршрути у місті як граф, де можна аналізувати ці маршрути між будь-якими точками.

У програмі реалізовано три незалежні алгоритми пошуку шляхів:

- **BFS (пошук у ширину):** знаходить найкоротший шлях у кількості ребер, але не враховує довжину дороги. Дає маршрут, який проходить через мінімум кількість проміжних вершин.
- **DFS (пошук у глибину):** повертає один із можливих маршрутів до мети, але не гарантує мінімальну кількість кроків. Характеризується тим, що спочатку йде “в глибину” — тобто вибирає один напрямок і досліджує його повністю.
- **Алгоритм Дейкстри:** знаходить найкоротший шлях з урахуванням ваг (довжини доріг). За допомогою `nx.all_pairs_dijkstra_path` і `nx.all_pairs_dijkstra_path_length` виводить найкоротші відстані між усіма парами районів. Цей модуль показує повну таблицю найкоротших шляхів і відповідних довжин.

Для аналізу вибрано маршрут: **Мій дім → Річковий вокзал**.

Порівняльна таблиця роботи роботи алгоритмів:

| Алгоритм   | Маршрут                                         | Кількість ребер | Загальна довжина (км) |
|------------|-------------------------------------------------|-----------------|-----------------------|
| BFS        | Мій дім → Центр → Місце роботи → Річковий вокзал      | 3               | 4+5+6=15              |
| DFS        | Мій дім → Ринок → Парк → Річковий вокзал        | 3               | 3+6+5=14              |
| Дейкстра   | Мій дім → Ринок → Парк → Річковий вокзал        | 3               | 3+6+5=14              |

**Висновки:**
BFS шукає шлях з мінімальною кількістю переходів, DFS — один з можливих (може не бути оптимальним), а Дейкстра — гарантовано найкоротший за сумою ваг.
